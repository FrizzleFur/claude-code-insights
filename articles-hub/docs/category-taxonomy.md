| 编号 | 名称 | 英文 | 说明 | 文章数 |

| C2 | 工程实践 | Engineering Practice | 编码最佳实践、上下文工程、工具设计 | ~6 |

## 难度等级

| 等级 | 名称 | 目标受众 | 特征 |
|------|------|---------|------|
| L1 | 入门 | 所有开发者 | 概念介绍为主，无需专业背景 |
| L2 | 中级 | 有 AI/LLM 经验的开发者 | 有具体技术方案，需要一定工程背景 |
| L3 | 高级 | 领域专家 | 深度技术细节，需要专业领域知识 |

## 内容形式

| 形式 | 说明 | 评分偏好 |
|------|------|---------|
| tutorial | 手把手教程，步骤清晰 | 可操作性和可复现性高 |
| conceptual | 概念框架，新视角新观点 | 创新性和教育价值高 |
| case-study | 实际项目案例分析 | 影响力和可操作性好 |
| reference | API/工具参考文档 | 实用性和时效性好 |
| research-paper | 学术论文风格 | 技术深度和创新性高 |

## 标签系统

**产品标签**: claude-code, mcp, sdk, api, desktop
**领域标签**: coding, research, customer-support, finance, legal
**模式标签**: prompt-chaining, routing, parallelization, orchestrator, evaluator, autonomous

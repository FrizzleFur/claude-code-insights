# 评分方法论

| 来源 | 用途 | 访问方式 |

## 评分流程

```
1. 获取文章全文
   └─> 通过 jina read_url 读取文章完整内容

2. 结构化提取
   └─> 识别核心概念、技术方案、代码示例、引用项目

3. 维度评分
   └─> 对照评分标准，逐维度打分（1-10）
   └─> 每个评分需有具体依据（引用文章中的段落）

4. 关联建立
   └─> 搜索关联 GitHub 项目
   └─> 建立文章间的前置/后续关系

5. 思维导图生成
   └─> 按「问题-方案-应用-扩展」四象限组织
   └─> 生成 Mermaid flowchart

6. 代码实践建议
   └─> 根据文章内容，提出 1-3 个可落地的实践项目
   └─> 标注难度等级和技术栈
```

## 评分一致性保障

### 基准文章

以下文章作为评分基准，新文章的评分应与基准文章对比:

| 基准文章 | 定位 | 预期综合分 |
|---------|------|-----------|
| Building Effective Agents | S 级标杆 | 9.0+ |
| Claude Code Quality Update | C 级参考 | 6.0-6.5 |

### 校准规则

1. **同一作者**的文章，技术深度评分不应差异过大（正负 2 分以内）
2. **同类别**的文章，影响力和可操作性应与该类别平均水平对比
3. **系列文章**（如 harness 系列），后续文章的创新性可能低于首发文章
4. **版本发布类**文章，时效性评分通常低于原则/方法论类文章

## 文章关系判定规则

### 前置关系 (prerequisites)

文章 A 是文章 B 的前置，当且仅当:
- A 中引入了 B 直接使用的概念或术语
- 不理解 A 的核心观点，就无法充分理解 B

### 后续关系 (follows_up)

文章 A 有后续文章 B，当:
- B 直接引用了 A 的内容或结论
- B 是对 A 中提出的问题的进一步探索

### 交叉引用 (cross_references)

- 文章 A 和 B 讨论相关但独立的话题
- 读者可以从任一篇开始阅读

## 数据更新策略

# 评分标准详解

## 7 维度评分体系

每篇文章从 7 个维度评分（1-10 分），通过加权计算得出综合评分。

### D1: 技术深度 (Technical Depth) — 权重 1.1

衡量文章的技术细节丰富程度和准确性。

| 分数 | 标准 | 示例特征 |
|------|------|---------|
| 1-2 | 表面描述 | 仅介绍概念，无技术细节 |
| 3-4 | 基础技术 | 提及技术方案但缺乏深度 |
| 5-6 | 中等技术 | 有具体的技术方案和实现思路 |
| 7-8 | 深度技术 | 详细的技术剖析、架构图、性能数据 |
| 9-10 | 前沿深度 | 深度技术剖析 + 代码示例 + 性能基准测试 |

### D2: 可操作性 (Actionability) — 权重 1.3 (最高)

衡量读者能否直接将文章内容应用到实际项目中。

| 分数 | 标准 | 示例特征 |
|------|------|---------|
| 1-2 | 纯理论 | 只有概念框架，无从下手 |
| 3-4 | 方向指导 | 提供方向但缺少具体步骤 |
| 5-6 | 部分可操作 | 有部分可执行的步骤 |
| 7-8 | 高度可操作 | 有完整的实施步骤和注意事项 |
| 9-10 | 即拷即用 | 可直接复制代码/配置到项目中使用 |

**为什么权重最高**: 作为学习项目，核心目标是"看完能做什么"，可操作性直接决定学习投资回报。

### D3: 创新性 (Novelty) — 权重 1.0

衡量文章是否提出新的概念、方法或视角。

| 分数 | 标准 | 示例特征 |
|------|------|---------|
| 1-2 | 常识复述 | 重复已有的公开知识 |
| 3-4 | 整合已有 | 将多个已有概念整合为系统性框架 |
| 5-6 | 新视角 | 对已有问题提供新的思考角度 |
| 7-8 | 新方法 | 提出新的方法论或技术方案 |
| 9-10 | 开创性 | 提出全新的概念范式（如 "Context Engineering"） |

### D4: 影响力 (Impact) — 权重 1.3 (最高)

衡量文章在社区中的传播广度和衍生影响。

| 分数 | 标准 | 示例特征 |
|------|------|---------|
| 1-2 | 小众关注 | 仅限特定领域关注 |
| 3-4 | 圈内关注 | 在 AI/LLM 开发者圈子有一定讨论 |
| 5-6 | 行业关注 | 被多篇技术文章引用 |
| 7-8 | 广泛影响 | 衍生出开源项目和社区实践 |
| 9-10 | 行业标杆 | 成为事实标准（如 "Building Effective Agents"） |

**衡量指标**:
- 直接影响: 衍生的 GitHub 项目数和 Star 数
- 间接影响: 社区讨论热度（HackerNews/Reddit/Twitter 讨论量）
- 引用频率: 被其他技术文章/书籍引用的次数

### D5: 教育价值 (Educational Value) — 权重 1.1

衡量文章对目标受众的学习帮助程度。

| 分数 | 标准 | 示例特征 |
|------|------|---------|
| 1-2 | 难以理解 | 表述模糊，缺少背景知识说明 |
| 3-4 | 需大量前置知识 | 需要很深的领域背景才能理解 |
| 5-6 | 有学习收获 | 中级开发者可以从中获得实用知识 |
| 7-8 | 高教育价值 | 有清晰的思维模型和层次化解释 |
| 9-10 | 思维模型级 | 改变读者思考问题的方式（如 Agent 设计原则） |

### D6: 时效性 (Timeliness) — 权重 1.0

衡量内容随时间贬值的速度。

| 分数 | 标准 | 示例特征 |
|------|------|---------|
| 1-2 | 短期价值 | 特定版本发布、临时配置 |
| 3-4 | 1年内有效 | 与当前版本紧密相关 |
| 5-6 | 1-2年有效 | 方法论层面，部分需要更新 |
| 7-8 | 长期有效 | 设计原则和架构模式 |
| 9-10 | 常青内容 | 基础概念和通用原则，几乎不会过时 |

### D7: 可复现性 (Reproducibility) — 权重 1.0

衡量读者能否按照文章内容复现相同效果。

| 分数 | 标准 | 示例特征 |
|------|------|---------|
| 1-2 | 无法复现 | 缺少关键步骤或参数 |
| 3-4 | 部分可复现 | 可以理解原理但无法完整复现 |
| 5-6 | 中等可复现 | 有示例但需要自行补充部分细节 |
| 7-8 | 高度可复现 | 有完整的代码示例和配置说明 |
| 9-10 | 完全可复现 | 提供完整的开源项目和文档 |

**参考基准**: html-effectiveness 项目 = 可复现性 9-10 分（20 个可直接运行的 HTML 示例）。

## 综合评分计算

```
overall_score = (
    technical_depth * 1.1 +
    actionability * 1.3 +
    novelty * 1.0 +
    impact * 1.3 +
    educational_value * 1.1 +
    timeliness * 1.0 +
    reproducibility * 1.0
) / 7.8  # 归一化到 10 分制
```

## 评分等级划分

| 等级 | 分数范围 | 含义 |
|------|---------|------|
| S 级 | 9.0-10.0 | 必读，行业标杆级内容 |
| A 级 | 8.0-8.9 | 强烈推荐，高价值内容 |
| B 级 | 7.0-7.9 | 值得阅读，有实用价值 |
| C 级 | 6.0-6.9 | 可选阅读，特定场景有用 |
| D 级 | < 6.0 | 价值有限，仅作参考 |
